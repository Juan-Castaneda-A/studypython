Programaci√≥n Orientada a Objetos (POO): Modela el Mundo

La POO es una forma de programar que te permite crear "moldes" para las cosas del mundo real. Es la base de casi todo el software moderno.

  • Una Clase es el "plano" o la "plantilla". Por ejemplo, la clase Coche.
  • Un Objeto es la "instancia" o la "cosa real" creada a partir del plano. Por ejemplo, mi_coche_rojo y tu_coche_azul son dos objetos de la clase Coche.
  • Los Atributos son las caracter√≠sticas del objeto (su color, marca, modelo).
  • Los M√©todos son las acciones que el objeto puede hacer (acelerar(), frenar()).
# Este es el plano (la Clase)
class Mascota:
    # El m√©todo __init__ es el "constructor", se ejecuta al crear un objeto
    def __init__(self, nombre, tipo):
        self.nombre = nombre # Atributo
        self.tipo = tipo     # Atributo
    
    # Este es un m√©todo (una acci√≥n)
    def presentar(self):
        print(f"Hola, soy un {self.tipo} y me llamo {self.nombre}.")

# Creas dos objetos (instancias) de la clase Mascota
mascota1 = Mascota("Max", "Perro")
mascota2 = Mascota("Manchas", "Gato")

# Usas sus m√©todos
mascota1.presentar() # Muestra: Hola, soy un Perro y me llamo Fido.
mascota2.presentar() # Muestra: Hola, soy un Gato y me llamo Misi.

Archivos (`.txt`, `.json`): Guardando Datos Permanentemente

Hasta ahora, toda la informaci√≥n de nuestros programas se pierde cuando los cerramos. Para guardarla, necesitamos escribirla en archivos. El formato JSON es perfecto para esto, porque funciona de manera muy natural con los diccionarios y listas de Python.

# diccionario de datos

# Guardar el diccionario en un archivo

    json.dump(datos_usuario, archivo)

# Ahora, cerramos y volvemos a abrir el programa...
# y podemos leer los datos desde el archivo
with open("datos.json", "r") as archivo:
    datos_leidos = json.load(archivo)
    print(datos_leidos["nombre"]) # Muestra: Ana
